关于Java开发、学习模式、与自制力

发表于 2021-02-13 分类于 学习探讨

Spring Boot

SpringMVC配置得快要抓狂，配置期间各种怀疑人生。配置文件多如牛毛，一直报错，却不知道错于何处。

但也并非全部是坏处，至少让我感受到了自己在技术方面的盲目自信，意识到技术是需要很庄重的对待的。是不可以短时间内掌握多门，每一门技术都需要花连续性的时间并坚持学习。

可能与我始终坚持用Idea有一定的关系，我笃定的认为在2020年 我无法接受像Eclipse这样操作逻辑混乱，UI丑陋的IDE。

今天配置SpringBoot成功了，idea似乎已经认为SpringMVC过时，在最新版本并没有给什么支持，反而是SrpingBoot项目给出了很多适配。

起初也是各种报错，观察了大佬的博客，使用一个YML的配置文件，几乎完成了所有的配置，我使用这个配置文件，成功一大步，配置文件越复杂，给程序员所造成的压力就越大，因为出错的可能性随着一个又一个文件而成倍数增加。这样来看，Django的确是最高性价比的Web开发框架，没有之一。因为几乎没有什么配置。逻辑上也很简单，视图、控制器、路由。

后来出现一个问题，在给Dao注入的时候出现的问题，我忘记怎么解决的了，这个小问题令我如鲠在喉，因为它已经足以决定成败。（可惜我忘记怎么解决的了）

过早的优化是一切烦恼的根源。技术最终的目的是为了使用，开发过程中有许许多多的不确定，过早的优化想法很大程度上只会使开发者心烦意乱

不得不说，JavaWeb配置的复杂，是Web开发者一道难以逾越的鸿沟。但是作为一名开发者，又不能局限于已经掌握的技术。对于未知不确定的痛苦，是不得不经受的。

学习模式

我们学习一门未曾谋面的课程的时候，会很自然的使用自底向上的学习模式，即掌握课程的个个模块，再来做拼装。

在技术的学习中，我发现自顶向下有着更加高效的学习效率，通过建立起最终学习成果的模型，然后再对个个模块的知识进行需求性的填充，不仅学习的目的性能动性加强，而且可以节省很多没有必要的学习。（因为可以感知深浅）

自制力

昨晚我渐渐才想明白，自由可以带来一些思想的高度独立，为人所不为，但是一味的“自由”与“放纵”的表现越来越像。

一味的遵从内心的欲望，对此不做干涉，凡事可为。可事有不可为，后果自品，如此与放纵无异。

对于其苦果我已不想再多做赘述，头晕、记忆力大幅度减退是家常便饭，碰上考试则是损失惨重。

最大的敌人是不加约束的自己。

目前成长路上所遇到的最大阻碍，就是不加约束！
